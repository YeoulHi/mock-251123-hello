# 제품 요구사항 문서 (PRD) - 채팅 애플리케이션

## 1. 제품 개요

본 문서는 구현 능력 향상을 목적으로 하는 채팅 애플리케이션의 요구사항을 정의합니다. 프로젝트의 핵심은 **채팅 서비스의 필수 기능(인증, 채팅방 관리, 메시징)을 최소한의 복잡도로 구현**하며, 구조적인 개발 방식과 상태 관리 패턴을 학습하는 데 있습니다.

상업적 목적, 화려한 UI/UX, 과도한 엔지니어링은 지양하고, `docs/goal.md`에 명시된 통과 조건을 만족시키는 것을 최우선 목표로 합니다.

## 2. Stakeholders

- **개발자**: 이 프로젝트의 핵심 이해관계자. 채팅 서비스 구현을 통해 기술적 역량을 강화하고, 깔끔한 코드와 아키텍처를 설계하는 경험을 쌓고자 합니다.
- **가상 사용자**: 개발자가 만든 애플리케이션을 사용할 가상의 최종 사용자. 이들은 원활한 채팅 기능을 기대합니다.

## 3. 포함 페이지

- **로그인 페이지**: 앱의 진입점으로, 미인증 사용자가 보게 될 첫 화면.
- **회원가입 페이지**: 새로운 사용자가 계정을 생성하는 페이지.
- **채팅방 목록 페이지**: 로그인한 사용자가 참여할 수 있는 모든 채팅방 목록을 보고, 새 채팅방을 생성할 수 있는 페이지.
- **채팅방 상세 페이지**: 특정 채팅방에 참여하여 다른 사용자와 메시지를 주고받는 페이지.

## 4. 사용자 여정 (User Journey)

### 여정 1: 신규 사용자의 회원가입 및 첫 채팅 경험

- **타겟 유저**: 서비스를 처음 방문한 신규 사용자
- **여정**:
    1.  사용자는 **로그인 페이지**에 처음 접속한다.
    2.  계정이 없으므로, 회원가입 링크를 통해 **회원가입 페이지**로 이동한다.
    3.  필요한 정보(이메일, 비밀번호, 이름)를 입력하고 가입을 완료하면 성공 다이얼로그를 확인한다.
    4.  자동으로 **로그인 페이지**로 돌아와 방금 만든 계정으로 로그인한다.
    5.  로그인 성공 후 **채팅방 목록 페이지**로 이동한다.
    6.  기존에 있던 채팅방 중 하나를 클릭하여 **채팅방 상세 페이지**로 진입하고, 첫 메시지를 보낸다.

### 여정 2: 기존 사용자의 채팅방 생성 및 참여

- **타겟 유저**: 계정이 있는 기존 사용자
- **여정**:
    1.  사용자는 **로그인 페이지**에서 로그인 후 **채팅방 목록 페이지**로 이동한다.
    2.  '새 채팅방 만들기' 버튼을 눌러 원하는 이름의 채팅방을 생성한다.
    3.  목록에 새로 생긴 자신의 채팅방을 확인하고 클릭한다.
    4.  **채팅방 상세 페이지**로 이동하여 대화를 시작한다.

### 여정 3: 채팅방 내 상호작용 (답장, 좋아요, 삭제)

- **타겟 유저**: 채팅방에 참여 중인 기존 사용자
- **여정**:
    1.  사용자는 **채팅방 상세 페이지**에서 다른 사용자가 보낸 메시지를 확인한다.
    2.  특정 메시지에 '답장하기' 아이콘을 눌러 해당 메시지를 인용하며 답장 메시지를 보낸다.
    3.  재미있는 메시지 옆의 하트 아이콘을 눌러 '좋아요'를 표시한다.
    4.  자신이 방금 보낸 메시지에 오타가 있는 것을 발견하고, '삭제' 아이콘을 눌러 메시지를 제거한다.

## 5. 정보 구조 (Information Architecture)

```
/ (Root)
├── /login (로그인 페이지)
│   - 로그아웃된 사용자의 기본 진입점
│
├── /signup (회원가입 페이지)
│
└── /(protected) - 인증이 필요한 경로 그룹
    ├── /chat (채팅방 목록 페이지)
    │   - 로그인된 사용자의 기본 진입점
    │
    └── /chat/[roomId] (채팅방 상세 페이지)
        - 동적 라우트, 특정 채팅방에 해당
```
